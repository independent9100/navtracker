# UKF Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an Unscented Kalman Filter (`UkfEstimator`) behind the existing `IEstimator` port, with a standalone testable sigma-point helper, and prove it correct by demonstrating it agrees with the EKF on linear measurements (the canonical sanity check for UKF implementations).

**Architecture:** Pure addition to `core/estimation/`. The UKF is a drop-in `IEstimator` alongside `EkfEstimator`; the harness can already evaluate it without any changes elsewhere. Sigma points + weights live in a separate small helper so they're testable in isolation. Reuses `predictMeasurement` / `measurementResidual` from `MeasurementModels` for h(x) and angle-wrapped residuals.

**Tech Stack:** C++17 · Eigen (incl. `Eigen/Cholesky` for `LLT`) · GoogleTest.

**Documentation standard (CLAUDE.md):** Each algorithm carries Math / Assumptions / Rationale / Ways-to-improve. Task 4 extends `docs/algorithms/estimation.md`.

## Design notes

- **Standard scaled UKF formulation.** Parameters `α, β, κ`. `λ = α²(n+κ) − n`; scale = `n + λ`. Sigma points: `χ₀ = x̂`; `χᵢ = x̂ ± (√(scale·P))ᵢ`. Weights: `w_m⁰ = λ/scale`, `w_c⁰ = λ/scale + (1−α²+β)`, all others `1/(2·scale)`.
- **Matrix square root via Cholesky** (`Eigen::LLT`). Cheap and standard; assumes `P` is positive-definite (true in steady state given PD `Q` and `R`).
- **Reuse existing measurement plumbing.** `h(x)` is `predictMeasurement(model, x).z_pred`; the Jacobian `H` is computed but discarded. Residuals use `measurementResidual` so bearing wrapping is consistent with the EKF.
- **The proof of correctness is the linear-agreement test.** On a linear `h`, UKF and EKF mean/covariance updates are mathematically identical. Asserting `t_ekf ≈ t_ukf` after the same input is the canonical UKF unit test.
- **Performance comparison deferred.** Comparing UKF vs EKF on the existing CV/position scenarios will show near-identical results (nonlinearity is mild). A meaningful comparison needs a scenario the EKF visibly degrades on (short-range bearing-only, sharp range/bearing geometry); that scenario hasn't been built yet and isn't part of this plan.

## Task 1: Sigma points + weights helper

**Math.** For state of size `n`, parameters `α, β, κ`: `λ = α²(n+κ) − n`, `scale = n+λ`. Cholesky `L Lᵀ = scale·P`. Sigma points: column 0 is the mean; columns 1..n are `mean + L.col(i)`; columns n+1..2n are `mean − L.col(i)`. Weights: `Wm₀ = λ/scale`, `Wc₀ = λ/scale + (1−α²+β)`, the rest `1/(2·scale)`.

**Assumptions.** `P` is positive-definite (Cholesky succeeds); `n+κ > 0` so `scale > 0`.

**Rationale.** Captures mean and covariance through nonlinear transforms more accurately than first-order linearization, at the cost of `2n+1` function evaluations.

**Ways to improve.** Square-root UKF (propagates `S = √P` directly for numerical stability); SVD-based square root for marginally-PD `P`.

```cpp
#include "core/estimation/SigmaPoints.hpp"

#include <Eigen/Cholesky>

namespace navtracker {

SigmaPoints computeSigmaPoints(const Eigen::VectorXd& mean,
                               const Eigen::MatrixXd& cov,
                               double alpha,
                               double beta,
                               double kappa) {
  const int n = static_cast<int>(mean.size());
  const double lambda = alpha * alpha * (n + kappa) - n;
  const double scale = n + lambda;

  const Eigen::LLT<Eigen::MatrixXd> llt(scale * cov);
  const Eigen::MatrixXd L = llt.matrixL();

  SigmaPoints sp;
  sp.points.resize(n, 2 * n + 1);
  sp.points.col(0) = mean;
  for (int i = 0; i < n; ++i) {
    sp.points.col(i + 1) = mean + L.col(i);
    sp.points.col(i + 1 + n) = mean - L.col(i);
  }

  sp.Wm.resize(2 * n + 1);
  sp.Wc.resize(2 * n + 1);
  sp.Wm(0) = lambda / scale;
  sp.Wc(0) = lambda / scale + (1.0 - alpha * alpha + beta);
  const double w = 1.0 / (2.0 * scale);
  for (int i = 1; i < 2 * n + 1; ++i) {
    sp.Wm(i) = w;
    sp.Wc(i) = w;
  }
  return sp;
}

}  // namespace navtracker
```

## Task 2: `UkfEstimator` (predict, update, initiate)

**Math.**
- **Predict.** Build sigma points from `(state, P)`. Propagate each through `F(dt)`. Recompute weighted mean and covariance; add `Q(dt)`.
- **Update.** Re-build sigma points from posterior-predict `(state, P)`. Propagate each through `h(model, ·)` to get `Zᵢ`. Predicted measurement mean `ẑ = Σ Wmᵢ Zᵢ`. Innovation covariance `S = Σ Wcᵢ (Zᵢ−ẑ)(Zᵢ−ẑ)ᵀ + R`, with bearing residuals wrapped. Cross-cov `Pxz = Σ Wcᵢ (χᵢ−x̂)(Zᵢ−ẑ)ᵀ`. `K = Pxz S⁻¹`; `x ← x + K (z − ẑ)` (residual angle-wrapped); `P ← P − K S Kᵀ`.
- **Initiate.** Same as EKF: position from `z.value(0..1)`, zero velocity, `P = blockdiag(R_pos, σ_v² I₂)`; status Tentative; MMSI hint copied to attribute if present.

**Assumptions.** Linear motion model `F(dt)·x` (CV); h(x) via `predictMeasurement`; `P` PD so Cholesky succeeds; `update` is called after `predict` advanced state to `z.time`.

**Rationale.** Matches the EKF's role at slightly higher cost (one `2n+1` × matrix multiply for predict, one for update). Wins materially when h is strongly nonlinear.

**Ways to improve.** Square-root UKF for stability; iterated UKF for stronger nonlinearity; analytic mean handling for bearings (circular statistics) when sigma spread is large.

- [ ] **Step 5: Create `core/estimation/UkfEstimator.cpp`**
```cpp
#include "core/estimation/UkfEstimator.hpp"

#include <utility>

#include <Eigen/Dense>

#include "core/estimation/MeasurementModels.hpp"
#include "core/estimation/SigmaPoints.hpp"

namespace navtracker {

UkfEstimator::UkfEstimator(std::shared_ptr<const IMotionModel> motion,
                           double init_speed_std,
                           double alpha,
                           double beta,
                           double kappa)
    : motion_(std::move(motion)),
      init_speed_std_(init_speed_std),
      alpha_(alpha),
      beta_(beta),
      kappa_(kappa) {}

void UkfEstimator::predict(Track& track, Timestamp to) const {
  const double dt = to.secondsSince(track.last_update);
  if (dt <= 0.0) return;
  const Eigen::MatrixXd F = motion_->transitionMatrix(dt);
  const Eigen::MatrixXd Q = motion_->processNoise(dt);

  const SigmaPoints sp =
      computeSigmaPoints(track.state, track.covariance, alpha_, beta_, kappa_);
  Eigen::MatrixXd prop(sp.points.rows(), sp.points.cols());
  for (int i = 0; i < sp.points.cols(); ++i) {
    prop.col(i) = F * sp.points.col(i);
  }

  Eigen::VectorXd mean = Eigen::VectorXd::Zero(track.state.size());
  for (int i = 0; i < prop.cols(); ++i) mean += sp.Wm(i) * prop.col(i);

  Eigen::MatrixXd cov =
      Eigen::MatrixXd::Zero(track.state.size(), track.state.size());
  for (int i = 0; i < prop.cols(); ++i) {
    const Eigen::VectorXd diff = prop.col(i) - mean;
    cov += sp.Wc(i) * diff * diff.transpose();
  }
  cov += Q;

  track.state = mean;
  track.covariance = cov;
  track.last_update = to;
}

void UkfEstimator::update(Track& track, const Measurement& z) const {
  const SigmaPoints sp =
      computeSigmaPoints(track.state, track.covariance, alpha_, beta_, kappa_);
  const int nz = static_cast<int>(z.value.size());

  Eigen::MatrixXd Zsp(nz, sp.points.cols());
  for (int i = 0; i < sp.points.cols(); ++i) {
    const MeasurementPrediction pred =
        predictMeasurement(z.model, sp.points.col(i));
    Zsp.col(i) = pred.z_pred;
  }

  Eigen::VectorXd z_pred = Eigen::VectorXd::Zero(nz);
  for (int i = 0; i < Zsp.cols(); ++i) z_pred += sp.Wm(i) * Zsp.col(i);

  Eigen::MatrixXd S = Eigen::MatrixXd::Zero(nz, nz);
  Eigen::MatrixXd Pxz = Eigen::MatrixXd::Zero(track.state.size(), nz);
  for (int i = 0; i < Zsp.cols(); ++i) {
    Eigen::VectorXd zd = Zsp.col(i) - z_pred;
    if (z.model == MeasurementModel::RangeBearing2D) zd(1) = wrapAngle(zd(1));
    const Eigen::VectorXd xd = sp.points.col(i) - track.state;
    S += sp.Wc(i) * zd * zd.transpose();
    Pxz += sp.Wc(i) * xd * zd.transpose();
  }
  S += z.covariance;

  const Eigen::MatrixXd K = Pxz * S.inverse();
  const Eigen::VectorXd y = measurementResidual(z.model, z.value, z_pred);
  track.state += K * y;
  track.covariance -= K * S * K.transpose();
  track.last_update = z.time;
}

Track UkfEstimator::initiate(const Measurement& z) const {
  Track t;
  t.last_update = z.time;
  t.status = TrackStatus::Tentative;

  Eigen::Vector4d x = Eigen::Vector4d::Zero();
  x(0) = z.value(0);
  x(1) = z.value(1);
  t.state = x;

  Eigen::Matrix4d p = Eigen::Matrix4d::Zero();
  p(0, 0) = z.covariance(0, 0);
  p(0, 1) = z.covariance(0, 1);
  p(1, 0) = z.covariance(1, 0);
  p(1, 1) = z.covariance(1, 1);
  const double vv = init_speed_std_ * init_speed_std_;
  p(2, 2) = vv;
  p(3, 3) = vv;
  t.covariance = p;

  if (z.hints.mmsi.has_value()) t.attributes.mmsi = z.hints.mmsi;
  t.contributing_sources.push_back(z.source_id);
  return t;
}

}  // namespace navtracker

## 4. Unscented Kalman Filter (`UkfEstimator`)

**Math.** Standard scaled UKF. For state dim `n` and parameters `α, β, κ`:
`λ = α²(n+κ) − n`; `scale = n + λ`. Sigma points:
`χ₀ = x̂`, `χᵢ = x̂ ± (√(scale·P))ᵢ` (Cholesky columns).
Weights: `Wm₀ = λ/scale`, `Wc₀ = λ/scale + (1−α²+β)`, others `1/(2·scale)`.

- Predict: propagate each `χᵢ` through `F(dt)`; reconstruct mean and
  covariance via `Σ Wmᵢ` / `Σ Wcᵢ` and add `Q(dt)`.
- Update: re-build sigma points from posterior-predict; propagate through
  `h(x)`; reconstruct predicted measurement mean `ẑ`, innovation covariance
  `S = Σ Wc(Zᵢ−ẑ)(Zᵢ−ẑ)ᵀ + R` (bearing residuals wrapped), cross-cov
  `Pxz = Σ Wc(χᵢ−x̂)(Zᵢ−ẑ)ᵀ`; `K = Pxz S⁻¹`; standard mean/cov update.
- Initiate: same one-point initiator as the EKF.

**Assumptions.** `P` is positive-definite (Cholesky succeeds); linear
motion model (`F(dt)·x`); `h(x)` continuous and bounded over the sigma
spread; sigma spread small enough that linearly averaging bearings around
`ẑ` is acceptable (true at operational geometry; not true near `±π`).

**Rationale.** Captures mean / covariance through `h(x)` more accurately
than EKF's first-order linearization, at the cost of `2n+1` evaluations of
`h` per update. Matters when nonlinearity is sharp (short-range range/bearing,
bearing-only). Verified correct by **exact agreement with the EKF on linear
measurements** (the canonical sanity check).

**Ways to improve / test next.** Square-root UKF (propagates `S = √P` for
numerical stability); iterated UKF; circular-statistics bearing mean when
sigma spread is wide; scenario where the EKF visibly degrades (short-range
bearing-only) to put a number on the UKF advantage.
````
