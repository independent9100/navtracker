## What was added to the diagnostic surface (additive, default-off)

`IPmbmDiagnosticSink` / `PmbmBernoulliDiag` (forensics hook, not a consumer
port) gained, per Bernoulli per scan:

- `innov_east_m`, `innov_north_m`, `innov_norm_m` — the TRUE applied-measurement
  **position innovation** of the dominant-hyp Bernoulli: (measurement ENU) −
  (predicted ENU, i.e. the parent's post-predict/pre-update mean), and its norm.
  Sentinel −1 when the Bernoulli misdetected / was born this scan / the claimed
  measurement carries no ENU position (bearing-only camera). Computed at the
  detection update in `enumerateChildren`, **only when a diagnostic sink is
  attached**, as a pure side-record that never re-enters the update math.
- `imm_mode_weights` — the per-mode IMM weights of the dominant-hyp Bernoulli
  (the Phase-2a declared (c) gap). Empty when single-Gaussian.

All Imazu/sim_ms radar+AIS and AutoFerry radar+lidar are `Position2D` (adapters
project range/bearing to ENU before the tracker), so the raw innovation is exact
for every measurement in the binding dying-track cases; only AutoFerry EO/IR
camera bearings are sentinel.

## Placement — where does the gate belong? (the question the proxy could not answer)

The innovation + IMM-mode timeline of the dying tracks is **not** a clean "one
oversized accepted innovation". It is two-phase (imazu_15 id 6 shown; imazu_22
id 1 and id 7 identical in shape):

```
  t     innov(m)  speed(m/s)  IMM [CV/CT]     phase
 315     18.1       2.3       0.94/0.06   ─┐  healthy, CV-dominant
 325     10.0      11.7       0.99/0.01    │  innovations 10–32 m
 327     67.0      33.8       0.24/0.76   ─┤  IMM flips to CT (turn) mode …
 335     10.8      42.0       0.12/0.88    │  … and speed climbs on
 337     53.5      75.1       0.07/0.93    │  MODERATE innovations (50–108 m,
 340     70.8     101.0       0.10/0.90    │  each < D_max) — the track grabs
 345     76.5     122.4       0.09/0.91    │  returns near its (fast) prediction
 350    105.2     ...         0.03/0.97   ─┘  so innovation stays small
 355    484.6     178.1       1.00/~0     ←── single OVERSIZED accepted innovation
                                              (gross mis-association); IMM slams
                                              CV↔CT; permanent runaway follows
```

`max innovation before the first >200 m flag`: id 6 = **108 m**, id 1 = 158 m,
id 7 = 173 m — all below D_max. So:

- **The velocity corruption STARTS with a sequence of moderate innovations**
  (50–108 m) under **IMM CT (turn) mode dominance** that pump the velocity state
  from ~2 to 100+ m/s over ~25 s. These are indistinguishable in magnitude from
  healthy innovations (early well-tracked scans carry 18–45 m), so a raw-
  innovation-magnitude gate **cannot** catch this phase.
- **A single oversized accepted innovation** (id 6: 484 m at t=355) — the gross
  mis-association — appears only *after* the velocity is already elevated. This
  the acceptance gate catches cleanly, with ~0 false-fire, before permanent loss.

**Placement verdict.** The clean, binding-passing TRIGGER is at
**update-acceptance** (the oversized innovation, D_max 200–400 m, ~0 false-fire).
But because the velocity is already elevated when it fires, a pure "reject the
update / coast one scan" ACTION would flag-but-not-fix (the corrupt velocity
just coasts on). The evidence points to pairing the acceptance trigger with a
**velocity-resetting action** (accept the position, reset/deweight the velocity
state) rather than a bare reject. The moderate-innovation build-up is an
estimator-internal effect (the CT-mode velocity update is too aggressive per
moderate innovation) — an estimator-side per-update velocity-change clamp is the
fallback if the acceptance-reset action alone does not recover the dying cases,
but note that a *magnitude* clamp on the buildup innovations would re-introduce
the Phase-2a velocity-signal false-fire, so it must key on the CT-mode /
velocity-change, not innovation magnitude.

### IMM-mode finding (closes the Phase-2a (c) gap)

The divergence coincides with the **CT (turn) mode taking over** (weight → 0.9+)
during the crossing ambiguity, then **violent mode thrashing** (weights slam
0↔1) once the oversized innovation lands. Healthy well-tracked scans sit at
CV-dominant (0.9+). CT-mode dominance is the divergence signature — supporting
the estimator-side reading of the build-up phase.
