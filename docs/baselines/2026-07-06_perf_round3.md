## Targets and outcomes

### T1 — one decomposition, not two (`S.determinant()` + `S.inverse()`)

Split per the arbiter ruling into two commits:

- **T1a (committed `f7b0914`, byte-identical / Class-A outcome):** the
  *scoring* sites — `IEstimator::logLikelihood` (base, used by EKF/UKF/PF) and
  `ImmEstimator::logLikelihood`. These feed only the association cost matrix;
  no assignment flips, so the outcome is **byte-identical** on all four gate
  sets AND both haxr workloads. (Eigen decomposes a *dynamic* 2×2 via LU
  already, so replacing `.inverse()`+`.determinant()` with one `PartialPivLU`
  is not even epsilon here — it is bit-for-bit.)
- **T1 remainder (state path):** `ImmEstimator::update` / `softUpdate` — folded
  into the T3 Class-B package below (they shift the recursive filter at fp
  scale and tip one borderline harbor decision, so they are priced with T3).

New helper `core/estimation/GaussianScore.hpp`: `gaussianScore` (solve + guarded
log-det, one LU), `luInverseDet` (explicit inverse + det, one LU, for sites that
reuse S⁻¹), and the fixed-size 2×2 kernels below.

### T2 — hoist measurement-independent prediction: BLOCKED BY DESIGN

The recompute T2 targets is in `PmbmTracker`'s cost/likelihood loop, which
reaches the estimator only through `IEstimator&`. Hoisting the per-mode
prediction across measurements would require a new port method (predict-once /
score-with-R) or coupling the tracker to IMM internals. **`IEstimator`'s surface
is shaped for algorithm swapping (the pluggable-hot-path invariant), not for
perf hooks; a predict-once/score-with-R method would leak the cost-loop's
batching strategy into every estimator implementation.** So T2 is *blocked by
design, benefit subsumed by T3* — with the 2×2 stack kernels each recompute is a
handful of stack flops (no heap), so avoiding the recompute buys ~nothing. Do
not re-propose the port change.

### T3 — fixed-size 2×2 stack kernels (committed `3fe5d1e`, Class-B package)

`ImmEstimator::gate` and `::logLikelihood` get a Position2D fast path: H = [I₂|0]
is a selector, so S = P[0:2,0:2] + R is built on the **stack** (`Eigen::Matrix2d`)
with a closed-form 2×2 score — no dynamic H, no matmul, no LU, no heap
temporaries per (track × measurement × mode). Other models keep the general
path. This is the folded H-elision (T2's linear-algebra insight) + fixed-size
kernels, and it removes the allocation churn the round-2 profile flagged.
Bundled with the T1 state-path remainder as one priced Class-B unit.
